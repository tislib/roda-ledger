**Key takeaways:**

- Async mode sustains **2.49M TPS** over 50s with sub-200ns queue insertion latency, processing 124M+ transactions with
  a steady ~30k–44k in-flight backlog.
- Wait mode reaches **1.92M TPS** (23% lower) while guaranteeing every sampled transaction is durably committed before
  returning, with P50 commit latency of **5.0ms**.
- The 56.2ms max in wait mode (first-second warmup at 18.1ms P99) reflects cold-start WAL/page-cache effects that settle
  within 1–2 seconds.
